Introduction à Markdown

Ce language a été créé en 2004 par John Gruber et Aaron Swartz et n’a pas évolué depuis. Même si de nombreuses extensions et “extras” sont venues se greffer au projet originel. Le projet initial avec sa documentation est sur daringfireball.

Son but est d'offrir une syntaxe facile à lire et à écrire. Un document avec des balises Markdown peut être lu en l'état sans donner l’impression d'avoir été balisé ou formaté par des instructions particulières.

Un document balisé par Markdown peut être converti en HTML, en PDF ou en d'autres formats. Bien que la syntaxe Markdown ait été influencée par plusieurs filtres de conversion de texte existants vers HTML, la source d’inspiration principale est le format du courrier électronique en mode texte.

Les fichiers sont généralement enregistrés avec l’extension .md (ou .markdown ) pour indiquer aux interpréteur la nature du texte qu’il vont lire; Mais ça n’a rien d’obligatoire.

Un document en Markdown permet d'implémenter une syntaxe, un script de conversion texte → HTML et un format de fichier.

Comme le résultat sera exporté en HMTL, vous pouvez tout à fait introduire directement des balises HTML dans votre texte ; mais celui-ci deviendra moins lisible et ne pourra plus être édité par quelqu’un ne maîtrisant pas le HTML. Attention, le formatage markdown ne sera pas appliqué à l’intérieur de ces balises.

Il est couramment utilisé pour les fichiers de documentation d’un projet ou d’un jeu de données (-souvent nommé readme.md), il est stocké au format texte classique et est plus léger que sa version interprétée puisqu’il ne contient pas les balises html.

La philosophie du système veut que le texte écrit soit lisible sans interpréteur particulier en mode texte.

Il est léger et épuré de l’essentiel de la verbosité d’un language balisé. Une fois converti, le navigateur web (qui joue alors le rôle d’interpréteur) en rendra la lecture plus claire.

Et c'est la tout l’intérêt du système : éditer un fichier texte simple avec quelques caractères spéciaux supplémentaires, puis le faire passer dans un script qui va l’enrichir des balises requises pour l’afficher avec une mise en forme dans le navigateur.

Il existe aujourd’hui de très nombreuses implémentations dans divers langages et des variantes augmentées de ce script pour générer du contenu lisible avec une mise en forme plus riche à partir du markdown.

Vous utiliserez au minimum celles de GitHub (pour les fichiers de documentation des projets) et Stack Overflow (pour écrire vos questions et vos réponses).

Exemples

Avec la syntaxe Markdown vous pouvez par exemple reproduire l'effet d'un tableau en html.

Tables Are Cool
col 3 is right-aligned $1600
col 2 is centered $12
zebra stripes are neat $1

Un autre exemple par ce qu'on ne va pas vous montrer que c'est possible sans vous vous expliquer comment :

| Tables        | Are           | Cool  |
| ------------- |:-------------:| -----:|
| col 3 is      | right-aligned | $1600 |
| col 2 is      | centered      |   $12 |
| zebra stripes | are neat      |    $1 |

Comme vous le voyez le code ci-dessus correspond a la syntax en markdown du tableau que nous avons réalisé plus haut.
